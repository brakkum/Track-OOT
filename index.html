"Custom logic" will use localy stored logic (generated with the logic editor).
The logic editor is a (hopefully) easy to use interface to generate or edit the logic of the tracker.
The editor was initially created for generating and editing rulesets for updates more easy,
but maybe someone wants to mess around with the logic a bit ;).

Currently skips are missing in logic, but I am on it, so stay tuned.
Randomizer settings are missing too, and, as soon as settings-strings go stable, parsing them.
